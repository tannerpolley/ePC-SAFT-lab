# Roadmaps

This folder holds full-scope, architecture, registry, and slice roadmap
documents. It is not the live milestone/issue mirror.

Use these documents for durable technical context:

- `FULL_ROADMAP.md` is the package-wide context and completion-standard roadmap.
- Architecture roadmaps describe technical doctrine and staged implementation gates.
- Registry files provide executable or generated roadmap evidence.
- Superseded slice plans may remain here when they explain historical package decisions.

Use `docs/milestones/` for local agent navigation of the current GitHub
milestones and open issues. GitHub Issues and the `ePC-SAFT Roadmap` Project are
authoritative for live tracker state.
