# epcsaft 0.2.0

`epcsaft` 0.2.0 starts the independent release line after the project was unforked.

## Install

This release note prepares the `v0.2.0` release. Publish the GitHub release only after final review.

After the GitHub release exists, Windows users can install the release wheel:

```powershell
python -m pip install C:\path\to\epcsaft-0.2.0-*.whl
```

PyPI publishing is configured through GitHub Actions. After the PyPI project page is live, use:

```powershell
python -m pip install epcsaft
```

## Changes

- Reset package metadata and documentation to version `0.2.0`.
- Cleaned Windows-first GitHub Actions for normal pushes, PRs, manual package checks, and PyPI Trusted Publishing.
- Updated CI smoke checks to use the current public API names: `ParameterSet`, `Mixture`, and `State`.
- Made hosted package builds explicit about no local Ipopt dependency unless an Ipopt install is configured.
- Rewrote the top-level README around the current package surface and truthful install paths.

## Publishing note

Publishing a GitHub release for `v0.2.0` triggers the PyPI Trusted Publishing workflow.
