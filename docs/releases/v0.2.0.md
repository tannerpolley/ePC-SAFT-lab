# epcsaft 0.2.0

`epcsaft` 0.2.0 is a Windows-first native-backed Python package release for PC-SAFT and electrolyte PC-SAFT thermodynamic calculations.

## Install

Windows users on CPython 3.13 can install the release wheel:

```powershell
python -m pip install C:\path\to\epcsaft-0.2.0-*.whl
```

The tagged source install builds the native extension locally for other supported Python versions:

```powershell
python -m pip install "epcsaft @ git+https://github.com/tannerpolley/ePC-SAFT.git@v0.2.0"
```

PyPI publishing is configured through GitHub Actions, but the first upload requires the PyPI pending publisher. After the PyPI project page is live, use:

```powershell
python -m pip install epcsaft
```

## Package Surface

- Provider-root exports: `ParameterSet`, `ModelOptions`, `Mixture`, `State`, and `create_input_template(...)`. Import `Equilibrium` from `epcsaft_equilibrium` and `Regression` from `epcsaft_regression` in the monorepo workspace.
- Runtime inspection is package-owned: `epcsaft.runtime_build_info()` and `epcsaft.capabilities()` report provider state, while the two extension packages report their own solver capabilities.
- `State` supports pressure closure, direct-density closure, `rho_guess` continuation, residual-property evaluation, fugacity/activity coefficients, and derivative result APIs.
- `Equilibrium(mixture, route=..., ...).solve()` is the public equilibrium workflow. The evidence-backed routes are `bubble_pressure`, `dew_pressure`, and scoped nonassociating hydrocarbon `single_component_vle` when the native Ipopt dependency is compiled.
- `Regression(mixture).fit_pure_neutral(...)` is the current `epcsaft_regression` workflow, backed by the native Ceres/CppAD path when those native dependencies are compiled.

## Boundaries

- Neutral LLE, electrolyte LLE, reactive speciation, reactive LLE, and reactive electrolyte LLE are declared not exposed by `epcsaft_equilibrium.capabilities()` for this release. Neutral LLE remains internal because its sampled-candidate Stage II audit is not a global HELD proof. Associating bubble/dew admission is limited to the source-backed Gross/Sadowski pressure-boundary fixtures reported by the capability evidence.
- The GitHub release wheel is built without a local Ipopt runtime dependency. Ipopt-backed equilibrium routes require an Ipopt-enabled source or editable build.
- Ipopt is an optional native dependency. Use explicit Ipopt-backed routes or solver options; `solver_backend="auto"` does not claim unvalidated Ipopt coverage.
- Source-checkout reference datasets are useful for examples and validation, but user projects should own their parameter data.

## Build And Release

- Release assets are built for the Windows CPython 3.13 no-local-Ipopt baseline.
- Hosted CI builds the package without local Ipopt unless an Ipopt install is configured.
- PyPI publishing remains a manual Trusted Publishing action after the pending publisher is configured.
