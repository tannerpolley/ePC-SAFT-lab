# epcsaft 0.1.0

`epcsaft` 0.1.0 is the first historical release receipt for the independent native-backed Python package.

## Historical note

This receipt corresponds to the retired `v1.5.0` GitHub release. No rebuilt package assets are attached.

## Highlights

- Native C++ ePC-SAFT runtime with a Python user API.
- Mixture/state construction through the original native adapter surface.
- Pressure closure, direct-density closure, `rho_guess` continuation, and density-consistency diagnostics.
- Fugacity coefficients, activity coefficients, residual properties, and runtime build/capability diagnostics.
- User-owned parameter-folder workflow.
- Native-backed regression and equilibrium work that fed the current capability registry.
- Source-checkout contracts for neutral equilibrium and additional route-family diagnostics that remain controlled by `capabilities()`.

## Notes

- The package metadata was prepared for PyPI under the name `epcsaft`, but the official download path was GitHub release/tag installation.
- Source-checkout reference datasets were useful for validation and examples; user projects were expected to own their parameter folders.
- Downstream applications owned their data, balances, run matrices, and acceptance criteria.
