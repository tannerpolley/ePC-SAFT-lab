# M0: Make Early Package PR Checks Advisory By Default

Ordinary early package-development PRs use local proof first. Heavy native,
package, release, and installed-provider lanes stay available but are
manual-only unless a PR explicitly claims release readiness, capability support,
or production native behavior.
