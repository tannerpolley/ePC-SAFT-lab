# ADR 0001: Architecture-Deepening Public API Boundaries

> Supersession note: ADR 0002 replaces the older `mixture.equilibrium(...)` convenience facade with direct `Equilibrium(mixture, ...)` and `Regression(mixture, ...)` public workflow constructors.

The package had drifted toward parallel descriptions of the same workflows: string-based equilibrium requests versus typed problem objects, canonical parameter records versus legacy runtime payloads, generic versus reactive target-family summaries, and hand-waved capability claims versus executable evidence. The original decision standardized internal typed `EquilibriumProblem` route inputs, `ParameterSet` as the canonical parameter-family boundary, `TargetDataset.target_family_summaries()` as the shared regression summary shape, and `epcsaft.capabilities()` as the public capability evidence surface. ADR 0002 then cut the public workflow interface over to direct `Equilibrium(mixture, ...)` and `Regression(mixture, ...)` constructors while retaining typed problem objects as internal route-adapter inputs until each route has focused public tests.
