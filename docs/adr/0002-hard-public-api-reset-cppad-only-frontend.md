# ADR 0002: Hard Public API Reset And CppAD-Only Frontend

The package is cutting over to a smaller public frontend instead of carrying legacy aliases, root free functions, backend-mode flags, typed problem root exports, or `Mixture` workflow factories indefinitely. Public provider users import `Mixture`, `State`, `Regression`, `ParameterSet`, `ModelOptions`, and `create_input_template` from `epcsaft`; equilibrium users import `Equilibrium` from `epcsaft_equilibrium`. `ParameterSet` stores ePC-SAFT parameter data only, while `ModelOptions` belongs to `Mixture` and selects formulation choices such as relative-permittivity mixture rules, SSM, DS, and Born variants. `Mixture` owns components, parameters, and model options only. `State(mixture, T=..., P=... or rho=..., x=..., phase=...)` owns thermodynamic conditions, while `Equilibrium(mixture, ...)` and `Regression(mixture, ...)` own workflow defaults and methods. Public `State`, `Equilibrium`, and `Regression` workflows must use CppAD-backed values and derivatives; exact Jacobians and Hessians are required for equilibrium and regression routes that the public API exposes. Existing analytic equation kernels may remain internal during the transition, but the reset frontend must not expose analytic, automatic, fallback, or user-selectable derivative modes. Missing CppAD coverage is therefore a blocking implementation gap, not a runtime option.
