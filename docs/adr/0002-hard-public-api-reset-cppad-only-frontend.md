# ADR 0002: Hard Public API Reset And CppAD-Only Frontend

The package is cutting over to a smaller public frontend instead of carrying legacy aliases, root free functions, backend-mode flags, and typed problem root exports indefinitely. Public users import `Mixture`, `State`, `Equilibrium`, `Regression`, `ParameterSet`, `ModelOptions`, and `create_input_template`; `ParameterSet` stores ePC-SAFT parameter data only, while `ModelOptions` belongs to `Mixture` and selects formulation choices such as relative-permittivity mixture rules, SSM, DS, and Born variants. Public `State`, `Equilibrium`, and `Regression` workflows must use CppAD-backed values and derivatives; exact Jacobians and Hessians are required for equilibrium and regression routes that the public API exposes. Existing analytic equation kernels may remain internal during the transition, but the reset frontend must not expose analytic, automatic, fallback, or user-selectable derivative modes. Missing CppAD coverage is therefore a blocking implementation gap, not a runtime option.
